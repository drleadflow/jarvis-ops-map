WHAT IS JARVIS

AI Employee Runtime

Not a chatbot. Not one giant agent. A foreman with workers, memory, checkpoints, tests, and approval gates. Jarvis receives tasks via Telegram or Claude Code, decomposes them, delegates to specialized workers, verifies results, learns from outcomes, and reports back.

Never loses the thread.

↓
EXECUTION MODEL

The Supervisor Loop

Every cycle: observe state, plan next step, act with tools, verify result, record state, decide to continue or escalate.

  • Load checkpoint from last known good state
  • Pick next task from queue (respecting dependencies)
  • Check policy (risk lane + budget)
  • Dispatch to appropriate worker
  • Verify result (tests, lint, type check, secret scan)
  • Save checkpoint and loop

↓
RETRY STRATEGY

4-Stage Recovery

  • Attempt 1: Same action, same strategy
  • Attempt 2: Alternative strategy (different approach)
  • Attempt 3: Fallback tool or simplified approach
  • Attempt 4+: Escalate to Christian with full diagnosis
BUDGET ENFORCEMENT

Resource Limits Per Project

  • Max steps: 100 (configurable)
  • Max tokens: 1,000,000
  • Max retries: 10
  • Max runtime: 24 hours
  • Warning at 80% consumed
  • Hard stop at 100%

PATTERN EXTRACTION

Learn From Every Run

After each worker completes a task, the extractor analyzes: what strategy was used, how long it took, token efficiency, whether retries were needed.

  • Success patterns: working strategy + worker type + approach
  • Failure patterns: error messages + anti-patterns to avoid
  • Correction patterns: retry sequences that eventually succeeded
  • Confidence scoring based on speed, token efficiency, retry count
↓
What promotion tier?
TIER 0 (Auto)Confidence >= 0.90, validated 3+ times, passes eval
TIER 1 (Staged)Confidence 0.70-0.89, needs review before promoting
TIER 2 (Human)External writes, credentials, compliance-related
↓
MEMORY RETRIEVAL

Before Similar Tasks

Query memory table for relevant patterns by tags, type, and content. Return top-N most relevant memories sorted by confidence. Inject into worker context.

  • Semantic search via pgvector embeddings
  • Tag-based filtering (worker type, project, tool used)
  • Confidence-weighted ranking
  • Stale pattern detection via evaluator
